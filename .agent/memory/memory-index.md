# Memory Index

This file is read FIRST at every session boot. It provides an overview of all memory files.
Detail files are loaded only on demand — this keeps token costs low and focus high.

## Core Memory Files

| File | Purpose | Load When |
|------|---------|-----------|
| [architecture-decisions.md](architecture-decisions.md) | ADR index — links to individual decisions in `adrs/` | Before any architectural change |
| [integration-contracts.md](integration-contracts.md) | API contracts, schema expectations, third-party assumptions | Before API/integration work |
| [session-handover.md](session-handover.md) | Unfinished work from last session | Every boot |
| [current-task.md](current-task.md) | Active task state (auto-generated by sync-task) | Every boot |
| [runbook.md](runbook.md) | Operational procedures, recurring commands | When doing operational work |
| [active-session.md](active-session.md) | Live session status (auto-updated) | Session tracking |
| [memory-index.md](memory-index.md) | This file — read first, then load details on demand | **ALWAYS first** |

## Boot Sequence

1. Read **this file** (memory-index.md)
2. Read `session-handover.md` for unfinished work
3. Read `current-task.md` for active task
4. Read `architecture-decisions.md` index (load individual ADRs on demand)
5. Read `integration-contracts.md` only if doing API/integration work
6. Load rules/workflows referenced above only when needed
