# Receiving Code Review Skill

**Source:** [Superpowers receiving-code-review](https://github.com/obra/superpowers/blob/main/skills/receiving-code-review/SKILL.md)

## When Receiving Review Feedback

1. **Read carefully** — Understand the actual concern
2. **Verify independently** — Check if the issue is real
3. **Decide** — Valid or not?

### If Valid → Fix + explain what changed
### If Not Valid → Push back with evidence

## Anti-Patterns

- ❌ "You're absolutely right!" without checking
- ❌ Implementing every suggestion blindly
- ❌ Defensive dismissal
- ❌ "Good catch!" without verifying

## Core Principle

**A review suggestion is a hypothesis, not a fact. Test it before accepting it.**
